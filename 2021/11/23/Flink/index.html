Flink简单学习

Flink主要侧重于流处理，Spark主要侧重于批处理。因此在后续工作中，Flink应该会是主流。

首先我们要了解流处理和批处理的区别。

流处理/实时计算 批处理/离线计算
数据大小未知 数据大小固定
简单操作 复杂操作
及时响应 需要一段时间
数据到达次序独立
再次提取数据代价大 方便查询计算结果

那什么是Flink勒？

Apache Flink是一个框架和分布式处理引擎，用于对无界和有界数据流进行有状态计算。Flink设计为在所有常见的集群环境中运行，以内存速度和任何规模执行计算。

当然了，在Flink中，上游task对下游task发送数据是有条件的

  • 数据达到32K(buffer)
  • 时间达到200毫秒

在Flink中，所有的task是一起启动的。

Spark和Flink任务调度方式(Flink和Spark都是粗粒度调度资源，即一开始就将资源全部申请下来)

Spark任务调度 Flink任务调度
1、构建DAG有向无环图 1、构建DataFlow
2、切分Stage 2、拆分成多个task
3、按照顺序将stage发送到taskScheduler 3、将所有task部署启动
4、taskScheduler将task发送到Executor中执行 4、等待数据流入并且处理数据
其中Spark是MapReduce模型 Flink是持续流模型

启动方式

yarn-session 在yarn里面启动一个flink集群 jobManager(ApplicationMaster)

先启动hadoop

yarn-session.sh -jm 1024m -tm 1096m

任务提交

和standallone一样。额外不同的是yarn中可以直接提交任务到yarn。命令如下

flink run -m yarn-cluster -yjm 1024m -ytm 1096m -c 任务名 上传的jar包

flink run -m yarn-cluster -yjm 1024m -ytm 1096m -c com.fst.core.Demo1WordCount flink-1.0.jar

  • yarn-session先在yarn中启动一个jobMansager ,所有的任务共享一个jobmanager (提交任务更快，任务之间共享jobmanager ， 相互有影响)

  • 直接提交任务模型，为每一个任务启动一个joibmanager （每一个任务独立jobmanager , 任务运行稳定）

Kafka的搭建

  • 修改config/server.properties
    • 1、broker.id=0，每一个节点broker.id 要不一样
      2、zookeeper.connect=master:2181,node1:2181,node2:2181
      3、log.dirs=/usr/local/soft/kafka_2.11-1.0.0/data 消息存放的位置
  • 启动
    • 先启动zk，因为kafka需要建立在zookeeper上。
    • 接着在每台节点启动broker，kafka是去中心化的架构 -daemon 后台启动 在所有节点启动
      • kafka-server-start.sh -daemon /usr/local/soft/kafka_2.11-1.0.0/config/server.properties
